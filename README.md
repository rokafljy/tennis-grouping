# 🎾 테니스 복식 대진표 만들기

테니스 정기모임에서 **참석자 이름만 넣으면** 코트별로 골고루 게임을 배정해 주는
복식(2:2) 대진표 자동 생성기입니다. 시합용 토너먼트가 아니라, 모두가 비슷한 횟수로
즐겁게 게임할 수 있게 미리 짜 두는 **친선 대진표**를 목표로 합니다.

## 이런 상황을 위해 만들었어요

- 복식 경기(한 코트당 4명, 2:2)
- 보통 2~3코트를 빌려서 20명 안팎이 함께 하는 모임
- 한 경기는 30분 이내(테니스 한 세트)
- 모임 시간 1~2시간 → 자동으로 2~4라운드로 나눠 배정

## 특징

- **공정한 경기 수** — 모든 참석자의 경기 수 편차가 최대 1 이내가 되도록 배정
- **다양한 파트너/상대** — 같은 사람과 반복해서 팀/상대가 되지 않도록 최소화
- **자동 휴식 배분** — 인원이 코트 수보다 많으면 라운드마다 골고루 돌아가며 휴식
- **복식+단식 자동 혼합** — 코트가 남고 인원도 남으면 복식을 우선 채우고 나머지를 단식으로 배정해 휴식을 최소화
- **레벨(A/B 그룹) 구분** — 앞쪽 라운드는 A끼리·B끼리만, 나머지 라운드는 전체를 섞어서 진행 가능
- **고정 매칭** — 특정 두 사람을 지정한 횟수만큼 반드시 같은 팀(파트너)으로 배정
- **모바일 친화 + 오프라인** — 코트에서 휴대폰으로 바로 사용, 서버 전송 없음
- **결과 공유** — 단톡방에 붙여넣을 수 있는 텍스트 복사 / 인쇄 지원
- **다시 섞기** — 마음에 안 들면 버튼 한 번으로 새 조합 생성

## 사용법

브라우저에서 `index.html`을 열기만 하면 됩니다. 빌드 도구가 필요 없습니다.

1. 참석자 이름을 한 줄에 한 명씩(또는 쉼표로 구분) 입력
2. 코트 수, 모임 시간(또는 라운드 수), 한 경기 시간 설정
3. **대진표 만들기** 클릭 → 라운드별 코트 배정과 휴식 인원이 표시됩니다

로컬에서 서버로 띄우려면:

```bash
npx http-server -p 8080 .
# 또는
python3 -m http.server 8080
```

## 대진표 배정 규칙

우선순위에 따라 배정합니다.

1. **경기 수 균등** (가장 중요) — 많이 뛴 사람이 먼저 휴식하도록 조정
2. **파트너 다양성** — 이미 팀이었던 조합은 피함
3. **상대 다양성** — 이미 상대였던 조합은 피함

매 라운드 뛰는 인원을 코트에 배정할 때, 여러 후보 조합을 만들어 위 규칙에 대한
"비용"이 가장 낮은 조합을 선택합니다. 같은 입력이면 항상 같은 결과가 나오며,
**다시 섞기**로 다른 조합을 얻을 수 있습니다.

### 레벨(A/B 그룹) 구분

참석자 입력에서 "레벨(A/B 그룹)을 구분해서 넣을래요"를 켜면 A그룹/B그룹을
따로 입력할 수 있고, "그룹별(레벨전) 라운드 수"로 앞쪽 몇 라운드를 그룹별로
진행할지 정할 수 있습니다.

- **그룹별 라운드**: 코트를 A/B 인원 비율로 나눠서, 같은 그룹끼리만 경기합니다
  (예: A10명·B10명·3코트 → A 2코트 + B 1코트).
- **혼합 라운드**: 그룹 구분 없이 전체를 섞어서 배정합니다.
- 경기 수·파트너·상대 반복 통계는 그룹 여부와 상관없이 전원에 대해 계속
  누적되므로, 그룹 인원이 달라 생기는 경기 수 차이도 혼합 라운드에서 자연스럽게
  보정됩니다.

### 고정 매칭

"특정 참석자끼리 파트너 횟수를 지정할래요"를 켜면, 한 줄에 `이름1, 이름2, 횟수`
형식으로 반드시 같은 팀으로 묶여야 할 쌍을 지정할 수 있습니다(횟수 생략 시 1회).
예: 부부·친한 친구 사이를 몇 번은 같은 팀으로 뛰게 하고 싶을 때 사용합니다.

- 요청한 횟수를 채울 때까지 배정 우선순위를 강하게 줘서, 최대한 빨리 매칭을
  성사시킵니다. 채운 뒤에는 다시 일반적인 "다양하게 섞기" 규칙으로 돌아갑니다.
- 지정 쌍이 많아질수록 나머지 인원의 파트너 다양성이나 전체 공정성에 미세하게
  영향을 줄 수 있습니다. 코트가 넉넉할수록 영향은 작습니다.
- 결과 화면에 각 쌍의 달성 현황(예: "2/2")이 표시됩니다. 라운드 수가 부족해
  물리적으로 다 채울 수 없는 경우 목표에 못 미칠 수 있습니다.

## 구성

| 파일 | 역할 |
| --- | --- |
| `index.html` | 화면 구조 |
| `styles.css` | 스타일 (다크/라이트 자동 대응) |
| `app.js` | 입력 처리·렌더링·복사/인쇄 등 UI 로직 |
| `scheduler.js` | 대진표 배정 알고리즘 (브라우저·Node 공용) |
| `scheduler.test.js` | 알고리즘 검증 스크립트 |

## 테스트

```bash
node scheduler.test.js
```

여러 인원/코트/라운드 조합에서 경기 수 편차가 1 이하인지, 같은 입력이 같은 결과를
내는지(결정성) 검증합니다.
